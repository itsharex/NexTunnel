- 后端导出给前端调用的方法（如 `GetTunnels`, `StartTunnel`）均定义在 `App` 结构体上，并返回标准化的错误类型或结构化数据。
- 内部模块（如 `tunnel`, `p2p`, `config`）普遍采用 `doc.go` 文件进行包级文档说明，保持代码自解释性。
- 前端 Store 层统一封装异步 API 调用，并通过 `extractErrorMessage` 辅助函数标准化错误处理逻辑。
- 配置与状态分离：持久化配置存储在 `internal/config`，而运行时连接状态由 `internal/tunnel/manager.go` 维护，并通过 `App` 层同步至前端。
