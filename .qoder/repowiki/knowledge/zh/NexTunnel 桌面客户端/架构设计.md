- **混合架构**：采用 Wails v2 框架，后端使用 Go 语言处理核心业务逻辑（隧道管理、P2P 引擎、配置持久化），前端使用 Vue 3 + TypeScript 构建用户界面。
- **后端分层**：`main.go` 和 `app.go` 作为 Wails 入口与绑定层，暴露 RPC 方法；`internal/tunnel` 负责隧道生命周期管理与协议交互；`internal/config` 基于 SQLite 实现配置持久化；`internal/p2p`、`internal/quic` 等模块提供底层网络能力。
- **前端结构**：`frontend/src` 包含视图组件（`StatusView.vue`）、状态管理（Pinia `tunnel.ts`）及 API 封装（`api/app.ts`），通过 Wails 运行时生成的绑定文件（`wailsjs/go`）调用后端方法。
- **数据流**：前端通过 Pinia Store 发起异步请求，经由 Wails 桥接调用 Go 端的 `App` 结构体方法，后者协调 `tunnel.Manager` 执行操作并更新 SQLite 数据库或内存状态。
