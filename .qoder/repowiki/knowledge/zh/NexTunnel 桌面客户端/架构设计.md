- **混合架构**：采用 Wails v2 实现 Go 后端与 Vue 3 前端的深度集成。Go 层通过 `embed.FS` 托管前端静态资源，并通过 `Bind` 机制将 `App` 结构体方法暴露给前端调用。
- **后端分层**：入口 `main.go` 初始化 Wails 运行时；`app.go` 作为业务协调层，管理 `tunnel.Manager` 生命周期及 SQLite 配置存储 (`internal/config`)；核心网络逻辑封装在 `internal` 包中，包括 P2P 引擎 (`internal/p2p`)、QUIC 传输 (`internal/quic`) 及 NAT 探测 (`internal/nat`)。
- **前端结构**：基于 Vite + TypeScript 构建，使用 Pinia 进行状态管理，Naive UI 作为组件库。通过自动生成的 `wailsjs` 桥接层调用后端 Go 方法。
- **依赖方向**：前端仅依赖 Wails 运行时接口；后端核心逻辑解耦为独立的 `internal` 模块，不直接依赖 UI 层。
