# 桌面端能力总览

NexTunnel 桌面端是普通用户的主入口。它把 Relay 连接、隧道创建、本机端口扫描、网络健康、Wintun 修复、运行日志和设置导入导出放在一个原生应用里。

## 页面结构

| 页面 | 用途 |
| --- | --- |
| 总览 | 连接 Relay、查看核心流量指标、实时流量图和最近日志 |
| 隧道 | 管理 TCP/HTTP 隧道，扫描端口并快速创建本地代理路由 |
| 网络 | 查看 NAT、P2P、真实 TUN、Wintun、虚拟网络路由和最近系统命令 |
| 日志 | 查看连接、隧道、设置、NAT、端口扫描、错误等活动日志 |
| 设置 | 管理服务端实例、网络参数、常用端口、安全、外观、通用设置和配置导入导出 |

## 连接服务端

1. 进入“设置 -> 连接”。
2. 新增服务端实例，填写 Relay、Control Plane 和 STUN。
3. 点击服务端检测，确认三类检查结果：
   - Relay TCP 可达性
   - Control Plane `/healthz`
   - STUN/NAT 探测
4. 回到总览页点击连接。

连接状态：

| 状态 | 含义 |
| --- | --- |
| 待机 | 未连接 Relay |
| 连接中 | 正在建立控制连接 |
| 已连接 | Relay 控制连接已建立，可以启动隧道 |
| 错误 | 地址、token、网络或服务端异常 |

## 隧道类型

桌面端支持两类本地代理：

| 类型 | 用途 |
| --- | --- |
| TCP | 暴露任意 TCP 服务，例如 SSH、数据库、本地开发端口 |
| HTTP | 暴露本地 HTTP 服务，适合 Web 开发和调试 |

隧道字段：

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| 名称 | `web-3000` | 本地唯一，便于 Dashboard 和日志识别 |
| 本地地址 | `127.0.0.1` | 当前仅建议填写回环地址或明确受控的本机地址 |
| 本地端口 | `3000` | 本机服务端口 |
| 远端端口 | `13000` | Relay 对外监听端口 |
| 类型 | `tcp` / `http` | 代理协议 |

## 连接类型展示

隧道列表会显示连接类型：

| 类型 | 说明 |
| --- | --- |
| P2P 直连 | P2P 引擎明确进入 connected 状态 |
| 中继 | 隧道运行中，但当前数据面通过 Relay |
| 待机 | 隧道未运行或未注册 |

v0.6.0-beta 以“真实运行态和限制说明”为准，不把未验证的系统路由 TUN 或 P2P 路径写成默认生产可用。

## 实时流量

总览页会定期读取桌面端运行态，并保存上传/下载流量快照。图表展示增量曲线，用于判断隧道是否有真实流量。

流量统计来自 Relay 连接和本地隧道管理器，适合快速观察，不替代服务端长期监控。

## 首次使用建议

1. 先在本机启动一个简单服务，例如 `127.0.0.1:3000`。
2. 在设置页保存服务端实例。
3. 在总览页连接 Relay。
4. 在隧道页创建 TCP 隧道并启动。
5. 通过远端端口访问本机服务。
6. 在日志页查看连接和隧道事件。
