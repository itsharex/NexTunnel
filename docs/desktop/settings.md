# 设置与多服务端实例

设置页按任务分区：连接、网络、端口、安全、外观、通用、关于。

## 连接

连接页用于管理服务端实例。每个实例包含：

| 字段 | 用途 |
| --- | --- |
| 节点名称 | 用户可读名称 |
| Relay 地址 | 桌面端连接的 Relay TCP 地址，例如 `example.com:7000` |
| Relay Token | Relay 客户端认证 token |
| Control Plane URL | 获取路由、注册节点和服务端检测的地址 |
| Control Plane Token | Control Plane Bearer Token |
| STUN Server | NAT 探测主服务器 |
| STUN Alt Server | NAT 探测备用服务器 |

服务端实例会持久化到本地 SQLite。切换实例后，总览页连接会使用当前实例。

## 服务端检测

检测会返回三类结果：

| 项目 | 检查内容 |
| --- | --- |
| Relay | TCP 地址是否可连接 |
| Control Plane | `/healthz` 是否可访问 |
| STUN | UDP STUN Binding 是否可返回公网映射 |

失败时会给出下一步建议，例如检查地址、token、UDP 出站策略或服务端防火墙。

## 网络

网络设置保存 STUN 和运行态摘要。完整网络诊断在 [网络健康与 TUN](./network.md) 页面。

## 端口

端口设置管理常用端口：

- 新增自定义常用端口。
- 启用或禁用内置端口。
- 删除自定义端口。
- 根据分类查看开发、数据库、服务、游戏和远程访问端口。

端口扫描仍只允许回环地址。

## 安全

安全分区展示：

- Relay Token 是否已配置。
- Control Plane Token 是否已配置。
- 本机端口扫描边界。

导出配置默认脱敏敏感字段。只有开启“包含敏感字段”时，才会导出 Relay Token 和 Control Plane Token。

## 外观

支持：

- 深色、浅色、跟随系统。
- 常规动效或减少动效。
- 品牌色选择。

减少动效会降低页面切换和微交互，适合系统启用 `prefers-reduced-motion` 的用户。

## 通用

通用设置包括：

- 界面语言：`zh-CN` / `en-US`
- 启动后自动连接
- 最小化到托盘
- 启动时最小化
- 开机自启
- 配置导入导出

导入配置会恢复设置和隧道。若导入文件中的敏感字段被脱敏，则保留当前已有 token。

## 关于

关于页展示当前版本、更新检查和诊断收集入口。诊断文本会避免暴露 token 字段名和敏感值。
